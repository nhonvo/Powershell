# Domain 1: Plan and Manage an Azure AI Solution (15-20%)

This domain focuses on setting up, securing, monitoring, and managing costs for Azure AI services, along with implementing Responsible AI frameworks.

## 🎯 Exam Objectives
- Select the appropriate Azure AI service or resource type.
- Configure security for Azure AI resources (keys, Entra ID, Managed Identities, network security).
- Monitor Azure AI services (diagnostic logging, metrics, alerts).
- Manage costs (pricing tiers, spending limits, scaling).
- Operationalize Responsible AI principles (bias detection, safety filters).

## 📖 Key Study Topics

### 1. Resource Provisioning
*   **Single-Service Resource**: Dedicated endpoint and key for a single service (e.g., *Azure AI Translator*). Better for billing isolation and fine-grained access control.
*   **Multi-Service Resource (Azure AI Services)**: Single key and endpoint for multiple services (Vision, Language, Translator, etc.). Simplifies key management. *Note: Azure OpenAI requires its own dedicated resource and cannot be accessed via a multi-service resource.*

### 2. Security & Networking
*   **Authentication**: API keys (primary/secondary for rotation) and Microsoft Entra ID (role-based access control - RBAC).
*   **Managed Identities**: Allows Azure resources (e.g., VMs, App Services) to authenticate to Azure AI services securely without hardcoding credentials.
*   **Network Security**: Restrict access using firewalls, virtual networks (VNs), and Private Endpoints (Private Link) to bypass the public internet.

### 3. Monitoring & Analytics
*   **Diagnostic Logs**: Stream logs to Log Analytics workspace, Azure Storage, or Event Hub to capture query inputs, output sizes, and caller IPs.
*   **Metrics**: Track latency, total transactions, errors (e.g., HTTP 429 Too Many Requests due to rate limits), and active connections in Azure Monitor.

### 4. Responsible AI
*   **Six Principles**: Fairness, Reliability & Safety, Privacy & Security, Inclusiveness, Transparency, and Accountability.
*   **Content Filtering**: Configure blocklists and severity thresholds (Low, Medium, High) in Azure OpenAI for categories like Hate, Sexual, Violence, and Self-harm.

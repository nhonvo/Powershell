# Module 7: PL/pgSQL & Stored Procedures 🧩

Khi logic nghiệp vụ quá phức tạp để viết bằng SQL thuần, chúng ta cần đến PL/pgSQL (Ngôn ngữ lập trình thủ tục).

---

## 🏗️ Kiến thức cốt lõi

### 1. Function vs Stored Procedure
- **Function**: Phải trả về giá trị, không thể chạy `COMMIT/ROLLBACK` bên trong. Dùng trong `SELECT`.
- **Procedure**: Có thể chạy `COMMIT/ROLLBACK` bên trong (Khuyên dùng từ Postgres 11+). Dùng lệnh `CALL`.

### 2. Triggers (Bẫy dữ liệu)
Tự động kích hoạt function khi có sự kiện `INSERT`, `UPDATE` hoặc `DELETE`.
- Ví dụ: Tự động cập nhật `updated_at` hoặc lưu lịch sử thay đổi `audit_logs`.

### 3. Cấu trúc Block
```sql
CREATE OR REPLACE FUNCTION update_log() RETURNS trigger AS $$
BEGIN
    NEW.updated_at := NOW();
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

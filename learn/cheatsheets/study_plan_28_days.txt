# Study Plan (28 Days) — Backend Engineer (.NET + AWS)

Based on your CV: **3+ years**, C#/.NET 8, AWS (ECS, SQS/SNS, DynamoDB, Aurora PostgreSQL), Redis, observability (Splunk/New Relic), CI/CD (CodeBuild/GitHub Actions), Hangfire, event-driven systems, integration testing, and GenAI (Bedrock + PGVector).

## How to use this plan

- Time budget: **2–3 hours/day** (weekday) + **4–6 hours/weekend**.
- Daily structure (recommended):
  - **Coding**: 60–90 min (DSA + patterns)
  - **Backend deep dive**: 45–60 min (design + .NET/AWS)
  - **Communication**: 15–30 min (behavioral + EN/JP speaking)
- Always write something: solutions notes, STAR story bullets, or a design write-up.

## Outcomes by Day 28

- **DSA**: 40–60 problems + strong pattern notes + timed practice.
- **System design**: 6–8 case studies written with tradeoffs.
- **Backend**: crisp answers for caching, queues, DB indexing, resiliency, testing.
- **Behavioral**: 12–15 STAR stories mapped to common questions.
- **Language**: stable 60s self-intro in **EN** and **JP**, plus common phrases.

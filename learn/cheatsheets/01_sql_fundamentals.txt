# 🐘 1. SQL Fundamentals (Core Mastery)

Nền tảng của bất kỳ cơ sở dữ liệu quan hệ (RDBMS) nào là ngôn ngữ SQL. PostgreSQL không chỉ tuân thủ chuẩn ANSI SQL cực kỳ khắt khe mà còn mở rộng nó với những kiểu dữ liệu mạnh mẽ như JSONB, Array, Range...

---

### 🏛️ 1.1 SQL Lifecycle (Vòng đời của Câu query)

Trước khi viết code, hãy hiểu PostgreSQL "nhìn" câu lệnh của bạn như thế nào:

1.  **Parser**: Kiểm tra cú pháp (Syntax). Nếu thiếu dấu `;`, lỗi sẽ trả về ngay.
2.  **Analyzer/Rewriter**: Chuyển câu lệnh thành một cây logic (**Query Tree**) và thay đổi các View/Rule cấu hình.
3.  **Planner/Optimizer**: (Bước Quan Trọng Nhất) Thử nghiệm hàng ngàn con đường khác nhau (Index Scan vs Seq Scan) và chọn cái tốn ít chi phí (Cost) nhất.
4.  **Executor**: Thực thi kế hoạch đã chọn và trả về kết quả từ ổ đĩa (Disk) hoặc RAM (Buffer Cache).

---

### 🧠 1.2 Kiểu dữ liệu (Data Types) Mastery

Chọn đúng kiểu dữ liệu giúp tiết kiệm 50% dung lượng ổ đĩa và tăng tốc độ query.

| Kiểu dữ liệu | Đặc điểm | Chọn khi nào? |
| :--- | :--- | :--- |
| **VARCHAR(n)** | Chuỗi có giới hạn độ dài. | Tên, Email, Địa chỉ. |
| **TEXT** | Chuỗi không giới hạn. | Mô tả sản phẩm, Nội dung bài viết. |
| **INTEGER / BIGINT** | Số nguyên 4 / 8 bytes. | ID, Số lượng, Điểm số. |
| **NUMERIC(p, s)** | Số thập phân chính xác tuyệt đối. | **Tiền tệ (Money)**, Tỷ giá hối đoái. |
| **TIMESTAMPTZ** | Thời gian có múi giờ. | Ngày tạo, Ngày cập nhật. (Luôn dùng loại có TZ). |
| **BOOLEAN** | True/False. | Trạng thái IsActive, IsDeleted. |

### 🚀 1.3 Tạo bảng (DDL) chuẩn Production

Đừng bao giờ tạo bảng mà thiếu các **Ràng buộc (Constraints)**.

**Ví dụ (Create Table):**
```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,        -- Khóa chính tự tăng
    username VARCHAR(50) UNIQUE NOT NULL, -- Không được trùng, không trống
    email TEXT UNIQUE NOT NULL,
    age INT CHECK (age >= 18),    -- Ràng buộc logic (Check)
    created_at TIMESTAMPTZ DEFAULT NOW() -- Tự động điền ngày tạo
);
```

---

### ⚡ 1.4 Thêm/Sửa/Xóa (DML) An toàn

#### **A. INSERT with ON CONFLICT (Upsert)**
"Nếu đã tồn tại thì cập nhật, nếu chưa có thì thêm mới." Cực kỳ hữu dụng khi đồng bộ dữ liệu.
```sql
INSERT INTO users (id, name) VALUES (1, 'Nhon')
ON CONFLICT (id) DO UPDATE SET name = EXCLUDED.name;
```

#### **B. DELETE with RETURNING**
Lấy lại dữ liệu vừa xóa mà không cần query lần nữa.
```sql
DELETE FROM users WHERE id = 1 RETURNING *;
```

### 📝 1.5 Bài tập thực hành & Thử thách (Practice)

**Bài 1**: [Lý thuyết] "Tại sao PostgreSQL lại khuyên dùng kiểu `NUMERIC` thay vì `FLOAT/REAL` để lưu trữ tiền tệ?"
<details><summary>Đáp án</summary>
`FLOAT` là số thực dấu phẩy động (Floating-point), độ chính xác không tuyệt đối (VD: 0.1 + 0.2 có thể ra 0.300000000004). Trong tài chính, 1 đồng cũng là tiền, `NUMERIC` đảm bảo tính toán chính xác 100% dù con số lớn đến đâu.
</details>

**Bài 2**: [Kỹ thuật] Viết câu lệnh `CREATE TABLE` cho một bảng **Sản phẩm (Products)** gồm: ID, Tên, Giá, Số lượng tồn kho, Ngày tạo. Hãy thêm ràng buộc: Giá luôn phải lớn hơn 0.
<details><summary>Solution</summary>
```sql
CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    name TEXT NOT NULL,
    price NUMERIC(15, 2) CHECK (price > 0),
    stock_qty INT DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```
</details>

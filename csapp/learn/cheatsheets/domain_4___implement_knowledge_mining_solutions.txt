# Domain 4: Implement Knowledge Mining and Document Intelligence (15-20%)

Covers configuring Azure AI Search to index, enrich, and retrieve unstructured and structured data.

## 🎯 Exam Objectives
- Provision and configure Azure AI Search.
- Create data sources, indexes, and indexers.
- Build custom search enrichment pipelines using Skillsets.
- Implement vector search and hybrid search.

## 📖 Key Study Topics

### 1. Azure AI Search Core Architecture
*   **Data Source**: Connection string to raw data (Azure Blob Storage, Cosmos DB, SQL).
*   **Index**: The searchable schema containing fields with properties: `searchable`, `filterable`, `sortable`, `facetable`, `retrievable`.
*   **Indexer**: Automates data ingestion by pulling data from the data source, running it through a skillset, and writing outputs to the index.

### 2. Enrichment Skillsets
*   **Skillset**: A collection of cognitive skills (OCR, Entity Recognition, Language Detection) that enrich raw document text during ingestion.
*   **Custom Web API Skills**: Integrates custom web endpoints (e.g., Azure Functions) into the indexing pipeline for custom business logic.
*   **Knowledge Store**: Directs enriched metadata outputs to Azure Tables, Blobs, or Files for external analysis.

### 3. Vector & Hybrid Search
*   **Vector Search**: Stores and queries text embeddings generated by embedding models (like OpenAI `text-embedding-3-small`). Queries semantic meanings instead of keywords.
*   **Hybrid Search**: Combines traditional BM25 keyword search with Vector search.
*   **Semantic Ranking (L2)**: Reranks the top results from a hybrid query using deep learning models to prioritize the most contextually relevant documents.

# Domain 5: Implement Generative AI Solutions (25-30%)

Covers using Azure OpenAI models for text, code, image generation, embeddings, and implementing Retrieval-Augmented Generation (RAG).

## 🎯 Exam Objectives
- Deploy and configure Azure OpenAI Service resources and models.
- Implement prompt engineering techniques (system prompts, few-shot learning).
- Use embeddings to perform semantic search or clustering.
- Build Retrieval-Augmented Generation (RAG) solutions using Azure OpenAI and Azure AI Search (On Your Data).
- Manage API rate limits, tokens, and Content Safety.

## 📖 Key Study Topics

### 1. Azure OpenAI Model Deployments
*   **Models**: GPT-4, GPT-4o, GPT-3.5-Turbo, DALL-E 3, Text-Embedding.
*   **Tokens**: The base units of text processed by LLMs. Billing is based on total input + output tokens.
*   **Rate Limits (TPM/RPM)**: Tokens Per Minute (TPM) and Requests Per Minute (RPM) limits are applied at the deployment level. Handling HTTP 429 (Too Many Requests) requires exponential backoff.

### 2. Prompt Engineering
*   **System Prompt**: Instructions injected at the start to define model behavior, tone, constraints, and safety guidelines.
*   **Few-Shot Learning**: Providing 1-3 concrete examples of input-output pairs in the prompt to guide formatting.
*   **Parameters**:
    *   *Temperature*: Higher values (e.g., 0.8) increase creativity/randomness. Lower values (e.g., 0) make outputs deterministic.
    *   *Top_P (Nucleus Sampling)*: Controls vocabulary pool based on cumulative probability. Choose *either* Temperature or Top_P, not both.

### 3. Retrieval-Augmented Generation (RAG)
*   **Concept**: Combines LLMs with external datastores (e.g., Azure AI Search index) to ground replies in specific source documentation and avoid hallucinations.
*   **Azure OpenAI 'On Your Data'**: Natively links Azure OpenAI to Azure AI Search, automating the retrieval, prompt injection, and response steps.
